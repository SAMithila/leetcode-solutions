## 📁 Structure

Solutions are organized by **pattern** rather than difficulty,
because recognizing patterns is the real skill.

| Category | Topics Covered |
|----------|---------------|
| [Arrays](./arrays/) | Hash maps, two pointers, sliding window |
| [Sorting](./sorting/) | Bubble, insertion, merge, quick, selection |
| [Searching](./searching/) | Linear search, prime numbers, divisors |
| [Recursion](./recursion/) | Base cases, fibonacci, factorial |

## 🔍 Approach

For each problem I focus on:
- **Brute force first** — understand the problem
- **Optimal approach** — identify the pattern
- **Time & space complexity** — O(n) analysis
- **Edge cases** — empty input, duplicates, negatives
